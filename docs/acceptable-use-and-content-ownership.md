# acceptable use and content ownership

## purpose

This project is for creators who want to sell original design resources directly to buyers and agents.

It is not built to host stolen content, bypass rights, or run abuse workflows.

## acceptable use

You may use this project to:
- publish original design systems, assets, templates, and related files;
- set pricing and payment requirements for your own content;
- deliver paid resources to users and agents through the documented API flow;
- run creator-owned commerce workflows with clear metadata and terms.

## prohibited use

Do not use this project to:
- upload or sell content you do not own or have rights to distribute;
- mirror, scrape, or republish copyrighted material without permission;
- run malware, userbot abuse, crypto mining, torrent aggregation, or illegal content distribution;
- provide open proxy or unrestricted file-hosting behavior outside this product scope;
- bypass payment requirements, receipts, or access controls.

## content ownership policy

You are responsible for all content you upload and sell.

By publishing content through this project, you represent that:
- you own the content, or
- you have explicit rights, licenses, or permissions to distribute it.

If you include third-party material, you must follow that material's license terms.

## licensing expectations

Each paid product should include clear usage terms. At minimum:
- who can use the content;
- what is allowed (commercial use, redistribution, modification);
- what is not allowed;
- attribution expectations, if required.

Recommended:
- include a `LICENSE.md` in each bundle;
- keep product-level terms in metadata or adjacent documentation.

## dmca and takedown handling

If a valid infringement claim is received, the disputed content should be removed or disabled while reviewed.

Repository operators should keep a simple process:
- acknowledge claim receipt;
- identify affected resource IDs/files;
- temporarily disable access;
- resolve ownership or licensing dispute;
- restore only when rights are clear.

## required support and dmca contact email

Any public reuse, deployment, or fork of this project must publish a working support/DMCA contact email.

Required placeholder (replace before launch):
- `support@yourdomain.com`

This email should appear in at least one public location:
- repository README,
- project website, or
- API documentation.

## security and abuse expectations

Operators should protect buyers and creators by:
- enforcing route-level access controls and payment checks;
- applying rate limits on paid endpoints;
- logging abuse indicators and failed payment attempts;
- rotating keys and keeping secrets out of source control.

## marketplace stance

This project is designed to support original creators with direct access to demand.

It intentionally offers an alternative to middleman marketplaces that control distribution and take a percentage of creator earnings.

## disclaimer

This document is a practical policy guide, not legal advice.

For high-value commercial use, consult legal counsel to finalize licensing terms and enforcement language.
