x402 paywalls for creator catalogs

Fork a minimal service for selling design files to agents.

Curatoria Starter gives creators a small Node service, a public catalog endpoint, and example product files so AI agents can discover paid design resources over HTTP.

Use the template Read the setup guide

Catalog first

Publish a machine-readable catalog at /.well-known/design-catalog.json with product names, prices, tags, and paid routes.

402 by default

Unpaid file requests return 402 Payment Required. Paid requests are verified through the x402 flow before files are delivered.

Bring your own work

Replace the example item under design-systems/, set your wallet address, test on Base Sepolia, then deploy your own instance.

Available today

Self-hosted starter path

  1. Fork or clone the starter template.
  2. Set WALLET_ADDRESS, ADMIN_API_KEY, and NETWORK.
  3. Add files and registry entries under design-systems/.
  4. Run npm run dev and confirm catalog plus 402 behavior.
